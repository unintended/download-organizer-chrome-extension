Download rules

The download rules are applied one by one whenever you start downloading a file. Each rule consists of matchers and a download path pattern. The matchers are actually a set of regular expressions that applied to the filetype, URL, referrer URL and filename of the download item. If all of the non-empty mathers succeed for some rule, the file is renamed according to the download path pattern of this rule, the other rules are skipped.

All the regular expressions are case-insensitive.

Matchers

You can use any RegExp constructions in the matchers (even the capturing groups).

MIME types

MIME filetype is a text string that describes a format of a file. Check the following list of the most common MIME types:
application/pdf
PDF documents
application/zip
Compressed archives
application/x-bittorrent
Torrent files
application/exe
Windows executable
audio/mpeg
MP3 files
image/jpeg
JPEG images
image/png
PNG images

Examples

image/.*
All images
application/(zip|rar)
Zip or rar archives
.*
Everything (catch-all)

Download path patterns

Download path consists of forward-slash delimited folders (e.g. my folder/my subfolder/). You can also specify the exact filename at the end (e.g. my folder/my file.zip). And you can use placeholders as dynamic values in the path.

Placeholders

${filename}
Original filename
${url}
Full download URL (before redirects)
${finalUrl}
Download URL after all redirects
${referrer}
Referrer URL
${tabUrl}
Current tab URL
${mime}
MIME type
${date}
Current date (YYYY-MM-DD format) or ${date:format} for custom formats

For ${date} placeholder you can use a date format as a parameter. E.g.: ${date:MMM DD, YYYY} or ${date:HH:mm}. You can use any format from the moment.js library.

Date format examples
YYYY-MM-DD
-
MM/DD/YYYY
-
MMM DD, YYYY
-
YYYY/MM
-
HH:mm

Capturing groups

If you use capturing groups (round brackets) in your matchers, you can reuse the matched groups in a download path by using ${field:1}, ${field:2}, etc. where field is a name of the matcher (mime, referrer, url, finalUrl, tabUrl or filename) and number is the index of the capturing group.

Examples

images/
Put all downloads to 'images' folder. Original filename will be appended automatically.
images/${filename}
Put all downloads to 'images' folder. Same as above.
downloads/${date:YYYY}/${date:MM}/
Organize stuff by year and month.
${referrer:1}/
Create a folder based on the first capturing group of referrer pattern (e.g. domain name).
